Analog Sound

Analog sounds should be prefixed by DP or suffixed by ~P (ReMooD Only).

ReMooD Data

ReMooD contains the bulk of the game definitions in the lump named REMOODAT. The primary data is contained within ReMooD.wad, while it will also be loaded from user supplied PWADs. ReMooD Data is also historically known as RMOD, which stood for ReMooD Map Object Data, however it no longer contains just data used for the map, it contains all of the data in a wide variety of areas. ReMooD Data replaces DeHackEd as the primary object modification method.

States

States are the bulk of what makes objects (MapObject) and weapons (MapWeapon) useful. Without states, you cannot see an object or make a weapon that has any use at all. States are never used standalone but only inside of other definitions.

Reference

  • DeHackEdNum [integer] -- DeHackEd state number identification. This is used for DeHackEd compatibility.
  • Tics [integer] -- Length of time the state lasts in 1/35th second intervals. A value of "-1" means this state lasts forever and a value of "0" means it instantly changes to the next state.
  • FastTics [integer] -- Similar to above, but when Fast Monsters is enabled this value is used instead.
  • Frame [integer/character] -- This is either a character matching the sprite frame number or a zero-based index of the sprite frame. That is, A is equal to 0, B is equal to 1, etc.
  • FullBright [boolean] -- Frame appears bright in the dark.
  • Sprite [string] -- A four character string that identifies the sprite to be displayed. Please note that this is used in touch specials (MapTouchSpecial) to identify the sprite to be displayed.
  • Priority [string] -- This determines how important the displayed frame is when viewed in the level. When the sprites in view limitation is reached an algorithm determines which sprite to replace, ones with lower priorities means they will not be visible. The possible values are: "Null", "Default", "Weapons", "Ammo", "WeaponFlash", "Effects", "Monsters", "Corpses", "Players", "Health", "Cookies", "MissionCritical", "Powerups", "Decorations", and "Projectiles".
  • Transparency [string/integer] -- This is how transparent the displayed sprite should appear when drawn. The possible values are numbers from "1" to "100" and special values such as "Fire" and "Effect1".
  • Function [string] -- This is the function to be execute when the state is set. This could be anything from monster actions to weapon actions. Each function varies based on its own use.
  • Goto [string] -- This indicates that the next state should jump to another state group. If this is specified with Next then this takes precedence.
  • Next [integer] -- This indicates the next frame to be displayed. If this is specified with Goto then this value is ignored.

Objects (MapObject)

Map objects define things that are in the actual map, that is the player, monsters, decorations, and other such things.

Weapons (MapWeapon)

This defines weapons, things that are used to attack enemies.

Reference

  • Ammo [integer] -- The type of ammo this weapon takes. Use "None" for infinite ammo.
  • NiceName [string] -- The appearance name of this weapon, this is used in obituaries and display names.
  • SBOGraphic [string] -- The picture to use when this weapon is used in the status bar overlay. The graphics usually start with "sbo".
  • SlotNum [integer] -- The slot number this weapon occupies. The values 1 thru 10 are permitted.
  • SwitchOrder [integer] -- The default switching order, this is used when using the previous and next buttons on the controller. This order is also used when switching based on slots. The built-in weapons are based around 100 to permit placing weapons in between.
  • NoAmmoSwitchOrder [integer] -- The order of weapons to switch to when you run out of ammo with the current gun. The weapon with the highest order will be switched to, provided there is enough ammo.
  • AmmoPerShot [integer] -- The amount of ammo used per shot. Note that this varies based on function. If you use the value 2 and create a weapon that fires a 3 round burst for example, 6 ammo will be used.
  • PickupAmmo [integer] -- Ammo received when this item is picked up.
  • DroppedObject [string] -- If a player dies and weapon drops are enabled, the player will drop this object.
  • GenericProjectile [string] -- Generic projectile to launch for the generic projectile function.
  • PuffClass [string] -- Uses another object class instead of the default ("BulletPuff") when this object hits the wall.
  • BringUpSound [string] -- This sound is played when the weapon is brought up.
  • IdleNoise [string] -- This sound is played when the weapon is in the ready state.
  • SpriteYOffset [floating] -- The Y offset of the weapon sprite when viewed by a player. This changes how high or low the weapon appears.
  • IsDoom [boolean] -- This weapon appears in Doom.
  • IsHeretic [boolean] -- This weapon appears in Heretic.
  • IsHexen [boolean] -- This weapon appears in Hexen.
  • IsStrife [boolean] -- This weapon appears in Strife.
  • IsNotShareware [boolean] -- This weapon cannot be used in shareware.
  • IsInCommercial [boolean] -- This weapon appears in commercial mode (aka Doom II).
  • IsRegistered [boolean] -- This weapon appears in non-shareware Doom and Heretic.
  • IsInExtended [boolean] -- This weapons appears in extended Heretic, The Ultimate Doom, and Doom 2.
  • IsBerserkToggle [boolean] -- This weapon is part of a berserker toggle. On older versions of Doom, the lowest ordered weapon with this flag is switched to when the player has the berserker powerup.
  • IsSwitchFromNoAmmo [boolean] -- Weapon is switched away when there is no more ammo remaining in this weapon. If this flag is not set then the weapon will not switch away when out of ammo.
  • IsStartingWeapon [boolean] -- The player starts with this weapon.
  • NoBleedTarget [boolean] -- When target is hit by a hitscan, it does not bleed.
  • NoThrust [boolean] -- This weapon does not push the target away from the player that fired it.
  • NoAutoFire [boolean] -- This weapon does not auto re-fire, this is used on the rocket launcher and BFG but a bug in the original Doom code prevents this from even working.
  • PrimaryBringUpState [states] -- These states are used when the weapon is brought up.
  • PrimaryPutDownState [states] -- These states are used when the weapon is lowered.
  • PrimaryReadyState [states] -- These states are used when the weapon is ready to fire.
  • PrimaryFireState [states] -- These states are used when the weapon is firing.
  • PrimaryFireHeldState [states] -- These states are used when the weapon is firing and the first fire sequence has been executed. This could be used for example with a flame thrower type weapon, the PrimaryFireState can be where the weapon was ignited and the PrimaryFireHeldState would be the actual flame shooting.
  • PrimaryFlashState [states] -- These states are used for weapon flashes.

Ammo (MapAmmo)

This defines ammo, which enable weapons to shoot.

Reference

  • ClipAmmo [integer] -- The amount of ammo per clip.
  • MaxAmmo [integer] -- The maximum amount of ammo that can be held for this ammo type.
  • StartingAmmo [integer] -- The amount of ammo the player starts with for this ammo type.

Touch Special (MapTouchSpecial)

Map touch specials define things that can be picked up (aka touched). This includes weapons, ammo, health, and armor on the ground along with any powerups floating or sitting around.

Skins

As in Doom Legacy, ReMooD supports custom player skins that can be used to change the avatar of the player. Skins are ASCII Text Files and are placed in lumps named S_SKIN, there may optionally be 2 characters after the name to help the author find them easier. Skins work in both Doom Legacy (and Doom Legacy based source ports such as ReMooD) and ZDoom based source ports. After the skin definition, if any custom sprites are used they should immediately follow the skin definition lump.

The following are fields that may be specified in the file, they are followed by equal signs ('=') then a value for the field.
Field
Description
name The skin will receive this name inside of the game and it will be identified by this name (in the profile menu). There is a maximum of 16 characters and this can only contain alpha-numeric characters and underscores, spaces are not allowed. If a name is omitted then it will receive a name of "Skin #" where # is a unique number. In ReMooD, if two skins contain the same name, the second skin's name will be mangled.
sprite The skin will use this sprite instead of the player's default sprite (PLAY). You must replace all frames of the sprite if you wish to use this feature.
face The skin will use different graphics for the status bar face. In Doom, they start with the letters STF, so if you are replacing STFST01, you would rename it to xxxST01 where "xxx" is the field value. You must replace all face graphics if you wish to use this feature.
dsoof
dsnoway
dsplpain
dspldeth
dspdiehi
dsslop
dspunch
dsradio
dsjump
dsouch		 None of these fields are required and you may choose any amount or order to replace. The field names correspond to the player sound you want to replace. All sound lumps must start with DS.

The following is a sample skin lump that will work in both Doom Legacy and ReMooD.
Sample Lump (S_SKINBR):
// Borg Drone Skin
name = borgdrone
sprite = BORG
face = BRG
dsnoway = dsbrg_no
dsplpain = dsbrg_pn
dspdiehi = dsbrg_hi
dsoof = dsbrg_of
dsslop = dsbrg_sl
dspunch = dsbrg_pu
dsradio = dsbrg_rd
dspldeth = dsbrg_dt
dsjump = dsbrg_jm
dsouch = dsbrg_ow
